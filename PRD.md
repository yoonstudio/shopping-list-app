# PRD: 사이드용 쇼핑 관리 앱 (Todo List App)

## 1. 프로젝트 개요

### 1.1 목적
매일 10-20개 정도의 쇼핑을 효율적으로 관리할 수 있는 사이드용 웹 애플리케이션

### 1.2 목표 사용자
- 사이드 사용자
- 일일 쇼핑 관리가 필요한 사람
- 간단하고 직관적인 도구를 선호하는 사용자

### 1.3 핵심 가치
- 단순성: 복잡한 설정 없이 바로 사용 가능
- 신뢰성: 데이터 손실 없이 안정적으로 동작
- 접근성: 브라우저만 있으면 어디서든 사용 가능

## 2. 기능 요구사항

### 2.1 핵심 기능

#### 2.1.1 쇼핑 추가
- 사용자가 새로운 쇼핑을 입력할 수 있는 입력 필드 제공
- 카테고리 선택 옵션 (메인/사이드/간식)
- Enter 키 또는 추가 버튼으로 쇼핑 등록
- 입력 검증: 빈 값 입력 방지

#### 2.1.2 쇼핑 수정
- 기존 쇼핑의 내용을 수정할 수 있는 기능
- 인라인 편집 또는 수정 모드 제공
- 카테고리 변경 가능

#### 2.1.3 쇼핑 삭제
- 개별 쇼핑 삭제 기능
- 삭제 전 확인 메시지 (선택적)
- 완료된 쇼핑 일괄 삭제 기능 (선택적)

#### 2.1.4 완료 체크 기능
- 체크박스를 통한 완료/미완료 상태 토글
- 완료된 항목의 시각적 구분 (취소선, 색상 변경 등)
- 완료 시간 기록 (선택적)

#### 2.1.5 카테고리 분류
- 3가지 카테고리: 메인, 사이드, 간식
- 카테고리별 시각적 구분 (색상 코딩)
- 카테고리별 필터링 기능
- 전체 보기 옵션

#### 2.1.6 진행률 보기
- 전체 쇼핑 대비 완료된 쇼핑의 비율 표시
- 진행률 바(Progress Bar) 또는 퍼센트 표시
- 카테고리별 진행률 표시 (선택적)
- 완료/전체 개수 표시 (예: 5/10)

### 2.2 데이터 관리

#### 2.2.1 데이터 저장
- localStorage를 활용한 브라우저 로컬 저장
- JSON 형식으로 데이터 저장
- 새로고침 시 데이터 자동 복원

#### 2.2.2 데이터 구조
```javascript
{
  id: "unique-id",
  title:  쇼핑
 내용",
  category: "메인|사이드|간식",
  completed: true|false,
  createdAt: "timestamp",
  completedAt: "timestamp" // 선택적
}
```

### 2.3 사용자 인터페이스

#### 2.3.1 레이아웃
- 단일 페이지 애플리케이션 (SPA)
- 상단: 제목 및 진행률 표시
- 중단: 쇼핑 입력 영역
- 하단: 쇼핑 목록 및 필터

#### 2.3.2 반응형 디자인
- 데스크톱과 모바일 환경 모두 지원
- 가독성 좋은 폰트 사용
- 적절한 여백과 간격

#### 2.3.3 사용자 피드백
- 작업 완료 시 시각적 피드백
- 에러 발생 시 알림 메시지
- 빈 상태(Empty State) 처리

## 3. 기술 요구사항

### 3.1 기술 스택
- HTML5: 구조
- CSS3: 스타일링
- Vanilla JavaScript: 로직 및 상호작용
- localStorage API: 데이터 저장

### 3.2 브라우저 호환성
- 최신 버전의 Chrome, Firefox, Safari, Edge 지원
- localStorage 지원 브라우저

### 3.3 성능
- 페이지 로딩 시간 1초 이내
- 쇼핑 추가/삭제 시 즉각적인 반응
- 100개 이상의 쇼핑도 원활하게 처리

## 4. 비기능 요구사항

### 4.1 사용성
- 직관적인 UI/UX
- 별도의 가이드 없이 사용 가능
- 키보드 단축키 지원 (선택적)

### 4.2 신뢰성
- 데이터 손실 방지
- localStorage 용량 초과 시 처리
- 예외 상황 처리

### 4.3 유지보수성
- 깔끔하고 읽기 쉬운 코드
- 주석을 통한 코드 설명
- 모듈화된 구조

## 5. 제약사항

### 5.1 기술적 제약
- 외부 라이브러리/프레임워크 사용 금지
- 순수 JavaScript만 사용
- 서버/백엔드 없이 클라이언트 사이드만 사용

### 5.2 데이터 제약
- localStorage 용량 제한 (일반적으로 5-10MB)
- 브라우저별 저장소 분리 (동기화 불가)

## 6. 개발 우선순위

### Phase 1: MVP (Minimum Viable Product)
1. 쇼핑 추가 기능
2. 쇼핑 목록 표시
3. 완료 체크 기능
4. localStorage 저장/복원
5. 기본 스타일링

### Phase 2: 핵심 기능 강화
1. 쇼핑 삭제 기능
2. 카테고리 분류
3. 진행률 표시
4. 카테고리별 필터링

### Phase 3: 개선 및 최적화
1. 쇼핑 수정 기능
2. 반응형 디자인 개선
3. 사용자 피드백 강화
4. UI/UX 개선

## 7. 성공 지표

- 사용자가 쇼핑을 추가하고 완료 체크할 수 있음
- 새로고침 후에도 데이터가 유지됨
- 진행률이 정확하게 표시됨
- 카테고리별 쇼핑 관리가 가능함
- 10-20개의 쇼핑을 원활하게 관리할 수 있음

## 8. 향후 확장 가능성 (Optional)

- 날짜별 쇼핑 관리
- 중요도/우선순위 설정
- 검색 기능
- 데이터 내보내기/가져오기
- 테마 변경 (다크모드 등)
- 알림 기능
- 반복 쇼핑 설정
