# 쇼핑 관리 앱 개발 - 단계별 프롬프트

이 문서는 PRD를 기반으로 한 5단계 개발 프롬프트입니다. 각 단계를 순서대로 진행하세요.

---

## 단계 1: 프로젝트 초기 설정 및 기본 구조

### 프롬프트:
```
project_03 디렉토리에 쇼핑 관리 앱의 기본 파일 구조를 만들어줘.

필요한 파일:
- index.html: 메인 HTML 파일
- css/style.css: 스타일시트 파일
- js/app.js: JavaScript 로직 파일

index.html 요구사항:
- HTML5 문서 구조
- UTF-8 인코딩
- 반응형 viewport 설정
- 제목: "쇼핑 관리"
- 다음 섹션 포함:
  1. 헤더: 앱 제목 표시
  2. 입력 영역: 쇼핑 입력 필드, 카테고리 선택 드롭다운(메인/사이드/간식), 추가 버튼
  3. 목록 영역: 쇼핑 목록을 표시할 컨테이너
- CSS와 JavaScript 파일 연결

기본 CSS 요구사항:
- 깔끔하고 모던한 디자인
- 중앙 정렬된 레이아웃 (최대 너비 600px)
- 적절한 여백과 패딩
- 색상 스키마: 밝은 배경, 진한 텍스트
- 버튼 hover 효과

순수 JavaScript만 사용. 외부 라이브러리 사용 금지.
```

### 예상 결과물:
- index.html (기본 구조)
- css/style.css (기본 스타일)
- js/app.js (빈 파일 또는 기본 구조)

## 단계 2: 쇼핑 추가 및 목록 표시 기능

### 프롬프트:
```
js/app.js 파일에 쇼핑 추가 및 목록 표시 기능을 구현해줘.

구현 요구사항:

1. 데이터 구조:
   - 각 쇼핑 객체: { id, title, category, completed, createdAt }
   - id는 Date.now() + Math.random()으로 고유하게 생성
   - 쇼핑 배열을 전역 변수로 관리

2. 쇼핑 추가 기능:
   - 입력 필드가 비어있으면 추가 방지 (alert으로 알림)
   - 추가 버튼 클릭 시 쇼핑 추가
   - Enter 키를 눌러도 쇼핑 추가 가능
   - 추가 후 입력 필드 초기화
   - 카테고리는 드롭다운에서 선택한 값 사용

3. 쇼핑 목록 표시:
   - 쇼핑이 추가되면 목록에 즉시 표시
   - 각 항목은 체크박스, 카테고리 뱃지, 쇼핑 내용을 포함
   - 쇼핑이 없을 때 "쇼핑이 없습니다" 메시지 표시

4. CSS 스타일링:
   - 각 쇼핑 항목은 카드 형태로 표시
   - 카테고리별 색상 구분:
     - 메인: 파란색
     - 사이드: 초록색
     - 간식: 주황색
   - 목록 아이템 간 적절한 간격

순수 JavaScript만 사용. DOM 조작은 querySelector와 createElement 사용.
```

### 예상 결과물:
- 쇼핑 추가 기능 동작
- 목록에 쇼핑 표시
- 카테고리별 색상 구분

## 단계 3: 완료 체크 및 localStorage 저장/복원

### 프롬프트:
```
완료 체크 기능과 데이터 영구 저장 기능을 추가해줘.

구현 요구사항:

1. 완료 체크 기능:
   - 각 쇼핑의 체크박스 클릭 시 completed 상태 토글
   - 완료된 쇼핑은 시각적으로 구분:
     - 텍스트에 취소선 적용
     - 텍스트 색상을 회색으로 변경
     - 약간 투명도 적용 (opacity: 0.6)
   - completedAt 타임스탬프 기록

2. localStorage 저장:
   - 쇼핑이 추가/수정/삭제될 때마다 localStorage에 저장
   - 키 이름: "todoList"
   - JSON.stringify()로 배열을 문자열로 변환하여 저장

3. 데이터 복원:
   - 페이지 로드 시 localStorage에서 데이터 자동 로드
   - JSON.parse()로 문자열을 배열로 변환
   - 데이터가 없으면 빈 배열로 초기화
   - 복원된 데이터로 목록 렌더링

4. 에러 처리:
   - localStorage 접근 실패 시 console.error로 로깅
   - try-catch로 JSON 파싱 에러 처리

순수 JavaScript만 사용. 모든 함수는 명확한 이름과 주석 포함.
```

### 예상 결과물:
- 체크박스로 완료 상태 토글
- 새로고침 후에도 데이터 유지
- 완료된 항목 시각적 구분

## 단계 4: 카테고리 필터링 및 삭제 기능

### 프롬프트:
```
카테고리별 필터링과 쇼핑 삭제 기능을 추가해줘.

구현 요구사항:

1. 카테고리 필터:
   - 쇼핑 목록 위에 필터 버튼 4개 추가: 전체, 메인, 사이드, 간식
   - 필터 버튼 클릭 시 해당 카테고리만 표시
   - 현재 선택된 필터는 시각적으로 강조 (active 클래스)
   - 기본값은 "전체"

2. 필터 로직:
   - 전체: 모든 쇼핑 표시
   - 메인/사이드/간식: 해당 카테고리만 필터링하여 표시
   - 필터링된 결과도 localStorage에 영향 없음 (표시만 변경)

3. 삭제 기능:
   - 각 쇼핑 항목에 삭제 버튼(×) 추가
   - 삭제 버튼 클릭 시 confirm으로 확인
   - 확인 후 배열에서 해당 항목 제거
   - localStorage 업데이트
   - 화면에서 즉시 제거

4. CSS 개선:
   - 필터 버튼 스타일링 (가로 나열, 적절한 간격)
   - 삭제 버튼은 쇼핑 항목 우측에 배치
   - 삭제 버튼 hover 시 빨간색으로 변경
   - 필터 active 상태 강조 (배경색 변경)

순수 JavaScript만 사용. 배열 메서드(filter, findIndex 등) 활용.
```

### 예상 결과물:
- 카테고리별 필터링 동작
- 삭제 기능 동작
- 필터 UI 및 삭제 버튼 스타일

## 단계 5: 진행률 표시, 수정 기능 및 최종 마무리

### 프롬프트:
```
진행률 표시, 쇼핑 수정 기능을 추가하고 전체적인 UI/UX를 개선해줘.

구현 요구사항:

1. 진행률 표시:
   - 헤더 아래에 진행률 섹션 추가
   - 진행률 바(Progress Bar) 표시
   - 완료/전체 개수 텍스트 (예: "5/10 완료")
   - 진행률 퍼센트 표시 (예: "50%")
   - 쇼핑이 추가/삭제/완료될 때마다 실시간 업데이트

2. 쇼핑 수정 기능:
   - 각 쇼핑 항목에 수정 버튼 추가
   - 수정 버튼 클릭 시 인라인 편집 모드로 전환
   - 입력 필드와 카테고리 선택 드롭다운 표시
   - 저장/취소 버튼 제공
   - 저장 시 데이터 업데이트 및 localStorage 저장

3. UI/UX 개선:
   - 반응형 디자인 (모바일 환경 고려)
   - 부드러운 애니메이션 효과 (항목 추가/삭제 시)
   - 빈 상태 메시지 개선 (아이콘 또는 이미지 추가)
   - 전체적인 색상 조화 및 가독성 개선
   - 그림자 효과로 입체감 추가

4. 최종 점검:
   - 모든 기능이 정상 작동하는지 확인
   - 코드에 명확한 주석 추가
   - 변수명과 함수명을 명확하게 작성
   - 중복 코드 제거 및 함수 모듈화

5. 진행률 바 CSS:
   - 배경색: 연한 회색
   - 진행 바 색상: 그라데이션 또는 단색 (초록/파랑)
   - 부드러운 전환 효과 (transition)
   - 높이: 20-30px, 둥근 모서리

순수 JavaScript만 사용. 최종 결과물은 완전히 동작하는 앱이어야 함.
```

### 예상 결과물:
- 진행률 표시 및 실시간 업데이트
- 쇼핑 수정 기능
- 반응형 디자인
- 애니메이션 효과
- 완성된 쇼핑 관리 앱
